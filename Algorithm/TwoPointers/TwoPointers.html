    # 双指针

    双指针顾名思义，是一种使用了两个指针的解法，一般用作数组中。

    # 基本概念

    本文介绍数组类的双指针问题，数组类的双指针问题，基本上分为三类：

    • 对撞指针，两个指针从相反方向移动
    • 快慢指针，一前一后的两个指针
    • 滑动窗口，一前一后的两个指针组成一个类似窗口的滑块移动

    其中，在快慢指针中核心介绍循环不变量的概念，基本上掌握了循环不变量的定义，就掌握了该解法。滑动窗口是一种特殊的快慢指针，但由于其特殊形式，所以单独构成一种类型。

    一般说来，快慢指针与滑动窗口的区别在于左指针是否会跳跃：

    • 在滑动窗口中，左指针不会跳跃，只会依次移动，窗口是逐渐滑动的
    • 快慢指针中，左指针是存在跳跃移动的可能性

    为了更好的介绍双指针概念，把双指针分为三个部分介绍：前言、对撞指针、快慢指针、滑动窗口。

    注意，本系列相关资料，大部分来源于力扣，一部分来源于网络。

    # 入门

    这里使用一个例题来做为双指针系列的入门。

    # LC 88. 合并两个有序数组 (opens new window)

    Q：

    给你两个按 非递减顺序 排列的整数数组 nums1 和 nums2，另有两个整数 m 和 n ，分别表示 nums1 和 nums2 中的元素数目。请你 合并 nums2 到 nums1 中，使合并后的数组同样按 非递减顺序 排列。

    注意：最终，合并后数组不应由函数返回，而是存储在数组 nums1 中。为了应对这种情况，nums1 的初始长度为 m + n，其中前 m 个元素表示应合并的元素，后 n 个元素为 0 ，应忽略。nums2 的长度为 n 。

    A：由于数组已有序，可以使用双指针，使时间复杂度达到O(m + n)

    /**
 * @param {number[]} nums1
 * @param {number} m
 * @param {number[]} nums2
 * @param {number} n
 * @return {void} Do not return anything, modify nums1 in-place instead.
 */
var merge = function (nums1, m, nums2, n) {
    // 双指针，尾插法，从后往前遍历
    let p = m + n - 1;
    let p1 = m - 1;
    let p2 = n - 1;
    while (p1 >= 0 && p2 >= 0) {
				// 寻找大的元素      
        if (nums1[p1] >= nums2[p2]) {
            nums1[p--] = nums1[p1--];
        } else {
            nums1[p--] = nums2[p2--];
        }
    }

    while (p1 >= 0) {
        nums1[p--] = nums1[p1--];
    }

    while (p2 >= 0) {
        nums1[p--] = nums2[p2--];
    }

};
