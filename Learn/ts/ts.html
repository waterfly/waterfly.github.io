    TypeScript 教程

    # TypeScript 教程

    记录 TypeScript 教程 (opens new window)中值得记录的点。并不完整记录TS语法，只记录认为值得记录的知识点。

    # 1. 简介

    JS属于动态语言，TS属于静态语言。

    静态语言的优点：

    • 有利于代码的静态分析
    • 有利于发现错误
    • 更好的 IDE 支持，做到语法提示和自动补全
    • 提供了代码文档
    • 有助于代码重构

    静态语言缺点：

    • 丧失了动态类型的代码灵活性
    • 增加了编程工作量
    • 更高的学习成本
    • 引入了独立的编译步骤
    • 兼容性问题

    因此，TS更适合更适合用在大型的企业级项目，不适合小型的、短期的个人项目。

    个人观点：对于项目应该尽量用TS，越使用越熟悉。

    # 2. 基本用法

    学习 TypeScript 需要分清楚“值”（value）和“类型”（type）。

    TS代码只涉及类型，不涉及值，所有跟值相关的都是JS处理。在TS项目中，只存在两种代码，一种是底层的值代码，使用JS语法；另一种是上层的类型代码，使用TS的类型语法。TS的编译过程，就是把类型代码全部移除，只保留值代码。

    TypeScript Playground (opens new window)，在线编译TS代码，可以直观的看到编译后的js代码。

    TS官方编译器叫做tsc，编译某个ts文件可以用tsc app.ts命令，编译成对应的js文件。

    TS编译的配置参数写在tsconfig.json文件中。

    ts-node (opens new window) 是一个非官方的 npm 模块，可以直接运行 TypeScript 代码。

    // 使用ts-node运行ts文件
ts-node script.ts

    # 3. any 类型，unknown 类型，never 类型

    类型继承关系图：

    知识点：

    1. TS中存在父类、子类概念，上面为父类，下面为子类
    2. 子类可以赋值给父类，父类不能赋值给子类。注意，any除外，准确的说any是表示关闭类型检查

    顶层类型有两个：

    • any，特殊类型，准确说any表示关闭类型检查，因此any可以赋值给任意类型，任意类型也都可以赋值给any。使用any会存在“污染问题”，因为关闭了对其的类型检查，就失去了TS的类型作用，因此一般用于js到ts的过度代码。

    • unknown，真正的顶端类型，任何类型都可以赋值给unknown，但是unknown不能赋值给除any以外的。一般来说，凡是需要考虑使用any的地方，都应该优先考虑使用unknown。unknown的类型变量使用场景是“类型缩小”，如下代码所示：

      let s:unknown = 'hello';
// 类型缩小，将一个不明确的类型缩小到明确的类型。
if (typeof s === 'string') {
  s.length; // 正确
}

    底层类型只有一个：

    • never，空类型，为了保持与集合论的对应关系，以及类型运算的完整性，引入了该类型。类型为空，不包含任何值。可以赋值给任意类型。（在集合论中，空集是任何集合的子集，因此TS规定，任何类型都包含了never类型）

      // 使用场景一，保证类型运算的完整性
function fn(x:string|number) {
  if (typeof x === 'string') {
    // ...
  } else if (typeof x === 'number') {
    // ...
  } else {
    x; // never 类型
  }
}

// 使用场景二，无任何返回值的函数
function f():never {
  throw new Error('Error');
}

let v1:number = f(); // 不报错
let v2:string = f(); // 不报错
let v3:boolean = f(); // 不报错

    # 4. 类型系统

    # 4.1 基本类型

    TS继承了JS的8种基本类型：

    • boolean
    • string
    • number
    • bigint
    • symbol
    • object
    • undefined
    • null

    # 4.2 包装对象

    在8种基本类型中，undefined和null是两个特殊值，object属于复合类型，剩下的5种基本属于原始类型，都有对应的包装对象（wrapper object）。包装对象：这些值在需要时，会自动产生的对象。

    五种原始类型：

    • boolean，通过 Boolean() 可以获取到包装对象。
    • string，通过 String() 可以获取到包装对象。
    • number，通过 Number() 可以获取到包装对象。
    • bigint，无法获取到包装对象，因为BigInt() 不能作为构造函数用。
    • symbol，无法获取到包装对象，因为Symbol() 不能作为构造函数用。

    注意，Boolean()、Stirng()、Number() 是JS内置的函数，用于将传入的参数转为对应类型的值，而 new Xxxx()是构造函数，用于创建一个新的Xxxx类型。

    包装对象实例：

    // 在JS中之后又对象才有方法，原始类型的值本身没有方法，因此在调用方法时，字符串会自动转为包装对象，charAt()方法其实是定义在包装对象上
'hello'.charAt(1) // 'e'

    由于包装对象的存在，每一个原始类型的值都有包装对象和字面量两种情况。

    'hello' // 字面量
new String('hello') // 包装对象

    为了区分这两种情况，TypeScript 对五种原始类型分别提供了大写和小写两种类型：

    • Boolean 和 boolean
    • String 和 string
    • Number 和 number
    • BigInt 和 bigint
    • Symbol 和 symbol

    ⚠️大写类型同时包括：包装对象和字面量两种情况；小写类型值包含字面量。

    // 大写类型包括：包装对象 和 字面量
const s1:String = 'hello'; // 正确
const s2:String = new String('hello'); // 正确

// 小写类型只包括：字面量
const s3:string = 'hello'; // 正确
const s4:string = new String('hello'); // 报错

    ⚠️建议：只使用小写类型，不使用大写类型。因为大部分使用类型，都是使用的小写类型，而且TS把内置的很多方法参数，都定义为小写类型，使用大写类型会报错。

    # 4.3 Object类型与object类型

    TS存在Object类型和object两种类型：

    • Object类型，广义对象，任何可以转成对象的值，都是Object类型，undfined和null不可以转成对象，空对象{}是Object类型的简写形式。
    • object类型，狭义对象，只包含对象、数组、函数。
    // Object 类型
let obj:Object;

// 正确
obj = true;
obj = 'hi';
obj = 1;
obj = { foo: 123 };
obj = [1, 2];
obj = (a:number) => a + 1;

// 错误，undefined和null不能转化为对象
obj = undefined; // 报错
obj = null; // 报错


// object 类型，只包含：对象、数组、函数
let obj:object;
 
obj = { foo: 123 };
obj = [1, 2];
obj = (a:number) => a + 1;
obj = true; // 报错
obj = 'hi'; // 报错
obj = 1; // 报错

    建议：总是只使用小写类型object，不使用大写类型Object。因为使用对象类型时，只希望包含真正的对象，不希望包含原始类型。

    # 4.4 undefined 和 null

    在JS中，undefined和null可以赋值给任何值，undefined表示还没有赋值，null表示值为空。在TS中，一般打开strictNullChecks来规避这种情况，打开后，undefinedhe和null就会遵守子类只能赋值给子类这一基本原则。

    # 4.5 值类型

    TS中，单一值也是一种类型，就是值类型。

    值类型一般是对应类型的子类型，比如'hello' 是string的子类型，4是number的子类型

    // x为值类型"hello"
let x : 'hello'

// x 的类型是 "https"，遇到const命令声明的变量，如果代码里面没有注明类型，就会推断该变量是值类型
const x = 'https';

// 值类型一般都是子类型
let x : 5 = 5;  //值类型，子类型
let y : number = 4 + 1;  //父类型，

x = y; // 错误，父类型 -> 子类型 错误
y = x; // 正确，子类型 -> 父类型 正确

    # 4.6 联合类型

    “类型缩小”是 TypeScript 处理联合类型的标准方法，凡是遇到可能为多种类型的场合，都需要先缩小类型，再进行处理。实际上，联合类型本身可以看成是一种“类型放大”（type widening），处理时就需要“类型缩小”（type narrowing）。

    function getPort(
  scheme: 'http'|'https'
) {
  switch (scheme) {
    // 类型缩小
    case 'http':
      return 80;
    // 类型缩小
    case 'https':
      return 443;
  }
}

    # 4.7 交叉类型

    交叉类型的主要用途是表示对象的合成。

    let obj:
  { foo: string } &
  { bar: string };

obj = {
  foo: 'hello',
  bar: 'world'
};

    # 4.7 type命令

    type命令用来定义一个类型的别名。

    // 定义别名
type Age = number;

let age:Age = 55;

    # 4.9 typeof运算符

    在JS中，typeof表示一元运算符，返回一个字符串，代表操作数的类型。在JS中，返回只能是八种基本类型名：

    typeof undefined; // "undefined"
typeof true; // "boolean"
typeof 1337; // "number"
typeof "foo"; // "string"
typeof {}; // "object"
typeof parseInt; // "function"
typeof Symbol(); // "symbol"
typeof 127n // "bigint"

    TS将typeof运算符移到了类型运算，操作数依然会是一个值，但是返回的不是字符串，是TS的类型。

    const a = { x: 0 };

// ⚠️返回的是类型，不是字符串！！！
type T0 = typeof a;   // { x: number }
type T1 = typeof a.x; // number

    因此，同一段代码中存在两种typeof运算符，一种是值相关的JS代码部分，另一种是类型相关的TS代码部分

    let a = 1;
// TS的typeof是类型运算
let b:typeof a;

// JS的typeof是值运算
if (typeof a === 'number') {
  b = a;
}

    即：

    • JS中的typeof，是值运算，返回是JS类型的字符串

    • TS中的typeof，是类型运算，返回的是TS的类型。在TS中，typeof运算符后面不能是表达式或类型

      // TS typeof 运算符不能是表达式，因为TS属于编译环节，编译时不会进行JS的值运算
type T = typeof Date(); // 报错

// TS typeof 命令的参数不能是类型
type Age = number;
type MyAge = typeof Age; // 报错

    # 5.数组

    # 5.1 数组基本写法

    TS数组有两种写法：

    • number[]，一般使用这种写法
    • Array，使用TS内置的Array接口，本质上属于泛型写法
    // 第一种写法
let arr:number[] = [1, 2, 3];

// 第二种写法
let arr:Array<number> = [1, 2, 3];

    # 5.2 数组中类型

    **TS的数组，成员类型必须相同。**如果数组内要使用不同的类型，一般有两种方案：

    • 使用any类型声明，数组内可以放任意类型。不推荐使用这种写法，失去了TS的价值。
    • 使用联合类型或元组声明，数组内可以放指定为联合类型的类型，推荐使用这种做法
    // 1. 使用any实现数组多类型，⚠️不推荐
const arr: any[] = [1, 'hello', true];

// 2.使用联合类型或元组
const arr: (number | string | boolean)[] = [1, 'hello', true];

    # 5.3 数组类型推断

    如果数组变量没有声明类型，TS 就会推断数组成员的类型。这时，推断行为会因为值的不同，而有所不同。

    // 1. 如果变量的初始值是空数组，则推断为 any[]
const arr = [];

// 2. 后续为这个数组赋值时，TS会自动更新类型推断
const arr = [];
arr // 初始没有赋值，推断为 any[]

arr.push(123);
arr // 推断类型为 number[]

arr.push('abc');
arr // 推断类型为 (string|number)[]


// 3. 类型推断的自动更新只发生初始值为空数组的情况。如果初始值不是空数组，类型推断就不会更新。
const arr = [123]; // 推断类型为 number[]
arr.push('abc'); // 报错

    # 5.4 只读数组

    TS中声明为只读数组后，不允许变动数组成员，删除、修改、新增数组成员都会报错。

    声明有三种写法：

    • 只用readonly关键字
    • 使用泛型只读数组ReadonlyArray<T>
    • 使用const断言 as const
    // 1. 使用 readonly 关键字
const arr:readonly number[] = [1, 2, 3];
arr[1] = 3; // 报错

// 2. 使用 ReadonlyArray<T>
const a1:ReadonlyArray<number> = [0, 1];

// 3。 使用 as const
const arr = [0, 1] as const;

    注意：只读数组 属于 数组的父类，即 父类：只读数组，子类：数组

    # 5.5 多维数组

    TS中使用T[][]的形式表示二维数组，T是 最底层数组成员类型，其他同上。

    var multi:number[][] =
  [[1,2,3], [23,24,25]];

    # 6. 元组

    元组（tuple）是 TypeScript 特有的数据类型，JavaScript 没有单独区分这种类型。它表示成员类型可以自由设置的数组，即数组的各个成员的类型可以不同。

    元组类型的基本写法：[number, string]，即类型写在数组内部。

    # 6.1 元组特点

    • 元组成员类型可以不同
    • 支持使用可选符（?）
    • 支持使用扩展符（...）
    • 元组成员支持添加成员名，用于说明，可以任意取名，没有实际意义
    • 支持只读元组，不允许修改（注意，只读元组是父类，元组是子类）
    // 数组，类型写在数组前面
let a:number[] = [1];

// 元组，类型写在数组内部，注意与数组的区别
let t:[number] = [1];
let x:[number, string] = [1, '1'];

// 元组，支持使用可选符（?），注意只能放到最后一个
let a:[number, number?] = [1];

// 元组，支持使用扩展符（...）	，支持在任意位置，此时元组数量会不固定
type t1 = [string, number, ...boolean[]];
type t2 = [string, ...boolean[], number];
type t3 = [...boolean[], string, number];

// 支持添加成员名，用于说明
type Color = [
  red: number,
  green: number,
  blue: number
];  // 表示RGB三原色，成员名只用来说明，没有实际意义
const c:Color = [255, 255, 255];


// 只读元素
// 写法一
type t = readonly [number, string]

// 写法二，使用工具类型 Readonly<T>
type t = Readonly<[number, string]>

    # 6.2 元组长度推导

    • 没有可选成员和扩展运算符时，TS会推导出元组的成员数量
    • 包含可选成员时，TS会推导出可能的成员数量
    • 包含扩展运算符时，TS无法推断出成员数量

    # 6.3 元组使用场景分析

    • 类似RGB三原色这种

    • 使用扩展符给函数传参

      {
  // 报错场景，因为对数组使用扩展符传参时，数量是不确定的
  const arr = [1, 2];
  function add(x:number, y:number){
    // ...
  }
  add(...arr) // 报错
}

{
  // 使用元组可以推导出元组数量，因此可以对元组使用扩展符传参
  const arr:[number, number] = [1, 2];
  function add(x:number, y:number){
    // ...
  }
  add(...arr) // 正确
}

    # 7. Symbol

    TS中存在symbol和unique symbol两种类型：

    • symbol，包含所有的Symbol值，但是无法表示某一个具体的Symbol值，即Symbol类型的总类型，有点类似number表示所有数字这个概念。
    • unique symbol，表示单个的、某个具体的Symbol值，不能被修改，只能使用const声明。是TS引入的类型，是symbol的子类型，类似于值类型的概念。unique symbol类型可以赋值给symbol类型。
    // 1. unique symbol 必须使用const声明
// 正确
const x:unique symbol = Symbol();
// 报错
let y:unique symbol = Symbol();

// 2. 使用const声明的，默认推导为 unique symbol类型
// 类型为 unique symbol
const x = Symbol();	
// 类型为 symbol
let x = Symbol();	
// 如果 const声明的变量赋值为另外一个symbol类型变量，则推导类型为symbol
let x = Symbol();
const y = x; // 类型为 symbol

// 3. symbol 是 unique syumbo的父类型，因此  unique symbo 可以赋值给 symbol 类型
const x:unique symbol = Symbol(); // unique symbol 类型
let y: symbol; // symbol类型
y = x; // unique symbol 赋值给 symbol类型，反之则报错

// 4. 每个unique symbol类型都是不同的，因此不能互相复制，要赋值只能使用 typeof
// 直接赋值会报错
const a:unique symbol = Symbol();
const b:unique symbol = a; // 报错
// 必须使用 typeof
const a:unique symbol = Symbol();
const b:typeof a = a; // 正确

    # 7.1 unique symbol类型作用

    1. unique symbol 类型的一个作用，就是用作属性名，这可以保证不会跟其他属性名冲突。如果要把某一个特定的 Symbol 值当作属性名，那么它的类型只能是 unique symbol，不能是 symbol。

      const x:unique symbol = Symbol();
const y:symbol = Symbol();

interface Foo {
  [x]: string; // 正确
  [y]: string; // 报错，属性值类型必须是unique symbol类型
}

    2. unique symbol类型也可以用作类（class）的属性值，但只能赋值给类的readonly static属性。

      class C {
  // unique symbol类型 只能赋值给类的 static readonly类型
  static readonly foo:unique symbol = Symbol();
}
