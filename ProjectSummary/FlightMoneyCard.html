    # 机票大赚卡需求总结

    # 1. 需求层面

    该需求整体较为简单，主要为 UI+一个接口，非常适合入门。 需求技术方案为：Vue + 万象，通过该需求熟悉了万象，熟悉了 Vue。 整体耗时约 8 天。

    # 2. 技术积累点

    # 2.1 Vue 知识点

    1. 子组件事件回传给父组件，核心为：this.$emit("父组件函数名");
    //父组件的写法
    <flightAlert
      @clickAlertLeft="clickAlertLeft"
    ></flightAlert>

    clickAlertLeft() {
        //to do somthing
    },

//子组件的写法
    leftButtonClick() {
      this.$emit("clickAlertLeft");     //通过 $emit 调用父组件函数名
    },
    1. 背景图片写法
      background: url("https://file.40017.cn/appresource/image/wangxiangyuejin/flightcoupon/top_bg_new.png")
    no-repeat;
  background-size: 100% 100%;
    1. board 图片拉伸写法，
      border-width: 270px 0px 185px 0px;
        border-style: solid;
        border-image-source: url("https://file.40017.cn/appresource/image/wangxiangyuejin/flightcoupon/coupon_bg.png");
        border-image-slice: 270 0 185 0 fill;
        border-image-width: 270px 0px 185px 0px;
        border-image-repeat: round;
        border-image-outset: 0;
    1. 伪元素 ::before、::after
    //对于：￥100这种情况，一般UI会让钱符号小一点，价格大一点，这种就可以使用伪元素来处理
.price-act {
  font-size: 64px;
  font-weight: bold;
  color: #ed5a55;
  margin-left: 38px;
  margin-top: 34px;
}
.price-act::before {
  content: "￥";         //伪元素的内容，用 content
  font-size: 36px;
  margin-right: -10px;
}
    1. attr()函数，用于伪元素 content 动态获取内容
    <p class="price-act" :priceOriginalSymbol="123">

.price-act::after {
  content: attr(priceOriginalSymbol);   //动态获取其他元素的属性值
}
    1. css 动画，呼吸灯效果
    
.pay-button {
  //这里声明需要使用动画
  animation: mybigsmall 1s linear infinite;
  -webkit-animation: mybigsmall 1s linear infinite;
}

//具体动画代码
@keyframes mybigsmall {
  0% {
    -webkit-transform: scale(1);
  }
  50% {
    -webkit-transform: scale(.9);
  }
  100% {
    -webkit-transform: scale(1);
  }
}

@-webkit-keyframes mybigsmall {
  0% {
    -webkit-transform: scale(1);
  }
  50% {
    -webkit-transform: scale(1.2);
  }
  100% {
    -webkit-transform: scale(1);
  }
}

    # 2.2 公司框架相关

    1. 分享相关，公司分享框架，看 PM 需求是否需要用特殊的唤起中间页背景，如果需要则让 UI 做特定尺寸的图片，使用后台进行配置，生成分享链接；如果不需要则用默认图生成即可。
    • 唤起中间页后台 (opens new window)
    //分享链接一般写法
//如遇微信被封的情况，则需要加token，token规则为十八味随机数，
//例如：shareToken: "_77480805547925e244",
      jumpUrl: `https://m.17u.cn/app/links/web/main${
          this.wanxiangConfig.shareToken
        }?url=${encodeURIComponent(
          location.href.split("?")[0] + `?refid=1752344963`

    1. 万象，万象分为组件和活动，活动为最后的活动页面，组件为 Vue 层面的组件。万象具备了配置功能，常用变动的文案或配置，可放到万象组件属性里，减少组件发版次数。

    # 2.3 工具

    • js-code-to-svg-flowchart (opens new window)，对 JS 文件生成函数调用关系图，但是还不完美，不能直接支持 Vue 文件，而且不能生成连续的调用图，后续改善。逻辑分散到几个函数以后，流程后面修改容易忘，需要一个整体的函数调用流程图，按照流程图来修改 Review 逻辑。
    • 代码统计工具 cloc (opens new window)
    //基本使用，跟代码所在目录
cloc ./
//
查看具体每个文件统计，跟--by-file
cloc ./ --by-file
    • Postman，需要详细研究，精通

    # 3. 小结

    1. 通过该项目熟悉了 Vue 的基本语法

    2. 通过该项目熟悉了万象的基本使用

    3. 该项目整体上还是非常适合入门

    4. 项目基本流程，分为以下接个阶段

      1. 策划稿分析，分析逻辑，确定技术选型，初步估时，估时一般按 1.5 计算
      2. UI 稿分析，分析页面 UI、动画难点，初步确定技术实现方案，确定项目真正估时，如果与初步估时不一致，则需要跟 PM、QA 同步，确定提测时间
      3. 接口分析，用 Postman 调试，分析接口节点是否满足，有不满足点沟通调整
      4. 页面开发，基于 UI 稿 + 接口 + 技术方案，开始实现
      5. 自测阶段，写一个覆盖大部分场景的测试案例自测，可用测试数据实现
      6. 提测阶段，修改 QA 反馈问题
      7. 上线
